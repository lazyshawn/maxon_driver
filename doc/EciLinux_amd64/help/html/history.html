Version History

  • Version: 1.11.3162.0, Date: 2017/12/12, OS: Linux amd64
    • added support for USB-to-CAN FD PCIe Mini and USB-to-CAR devices.
    • updated LIN core for CAN-IB interfaces to support LIN wakeup messages.
    • ECI_LIN_MSGTYPE_SLEEP message type introduced.
    • Updated CAN-IBxxx firmware to version 0.3.11.0.
    • EciDeviceAdmin - Added USB-to-CAN FD V0.1.0.3 firmware.
    • EciDeviceAdmin - Updated CAN-IB200 fpga to V0.1.8.0 and boot manager to V0.3.0.8
    • EciDeviceAdmin - Updated CAN-IB600 fpga to V1.1.4.0 and boot manager to V0.3.0.8
    • EciDeviceAdmin - Added USB-to-CAN FD V0.1.0.3 firmware.
  • Version: 1.11.3074.0, Date: 2017/06/16, OS: Linux i386 and Linux amd64
    • removed libusb dependency from unneeded driver libraries.
    • corrected CAN-IB530 and CAN-IB630 device id's.
    • kernel module bug fix on IOCTL_GETDEVID calls if there is no pci_dev->bus set domain_nr to 0.
    • kernel module bug fix eci_read_info memory access violation.
    • Moved dynamic inline functions into class implementation as they code compiled to static code.
    • Bug fix in ECI_CAN_OPMODE_STT for CAN-FD communication.
    • For USB-to-CAN devices don't reset configuration if the desired one is the current on.
  • Version: 1.11.3019.0, Date: 2016/12/14, OS: Linux i386 and Linux amd64
    • Added support for USB-to-CAN FD family.
    • Allow LIN controllers to send ECI_LIN_MSGTYPE_WAKEUP messages.
    • ECICTRL_GetHwInfo returns the requested struct version number.
    • Updated predefined CAN-FD bittimings for 8 Mbit/s and 10 Mbit/s.
    • EciDeviceAdmin - Updated USB-to-CAN V2 firmware to version 1.5.8.
    • VxWorks 6.9.x - Implemented driver libraries and examples for real time process applications.
    • VxWorks 6.9.x - Replaced POSIX threads, mutexes and events with VxWorks Traditional functions.
    • VxWorks 6.9.x - Added work around for opening eci char devices on network file systems.
  • Version: 1.11.2841.0, Date: 2016/04/29, OS: Linux i386 and Linux amd64
    • Added VxWorks 7 support.
    • Added support for CAN-IB110/XMC Card, CAN-IB310/PMC Card, CAN-IB700/PMC Card, CAN-IB510/XMC, CAN-IB210/XMC, CAN-IB410/PMC, CAN-IB610/XMC and CAN-IB810/PMC interfaces.
    • Replaced kernel module's flush method with release and allow it to open the char device several times for the owner process.
    • Replaced 'check_mem_region' function calls with 'request_mem_region'
    • Added 32-bit compatibility mode for amd64 systems within the kernel module. Note due to this, old versions of the kernel module are incompatible with this driver libraries.
    • Implemented ECI_CAN_OPMODE_CACHED_STATUS and ECIDRV_CtrlSetStatusUpdateRate().
    • Updated 'End User License Agreement'.
    • Implemented ECI_HW_INFO version 1 which holds the device id.
    • Bugfix of timer overrun storm in XC16 PCI/PCIe interfaces.
    • Updated fpga to 1.1.3.0 for CAN-IB500 PCIe and CAN-IB520 PCIe Mini interfaces.
    • Updated fpga to 1.1.2.0 and bootmanager to 0.3.0.6 for CAN-IB600 PCIe interfaces.
    • Updated fpga to 0.1.1.0 for CAN-IB700 PCI interfaces.
    • Updated fpga to 0.1.1.0 and bootmanager to 0.3.0.5 for CAN-IB800 PCIe interfaces.
  • Version: 1.10.2640.0, Date: 2015/03/18, OS: Linux i386 and Linux amd64
    • Added support for CAN-IB5x0/PCIe FD, CAN-IB600/PCIe FD, CAN-IB700/PCI FD and CAN-IB800/PCI FD interfaces.
    • Added CAN-FD support to ECI-API for more information see ECI_CANINITLINE, ECI_CANMESSAGE, ECI_CANCAPABILITIES and ECI_CANSTATUS
    • CAN-FD bittimings improved
    • Updated firmware to 0.3.4.2 for iPC-I XC16/PCI(e) series interfaces to support LIN Error Frame Mode.
    • Updated firmware to 0.3.9.3 for CAN-IB2x0/PCIe, CAN-IB400/PCI, CAN-IB600/PCIe FD and CAN-IB800/PCI FD series interfaces
    • LinuxEciDeviceAdmin - added ISO CAN FD firmware for CAN-IB500 and CAN-IB600 PCIe interfaces with HW Version 1.1.1.0 and Boot Manager Version 0.3.0.5
    • Increased ECI_MAXBOARDCOUNT from 8 to 32 supported boards
  • Version: 1.9.2493.0, Date: 2014/11/18, OS: Linux i386 and Linux amd64
    • Added support of USB-to-CAN V2 for QNX 6.5, QNX 6.6, VxWorks 6.9.3 and INtime 5.02
    • Added VxWorks 6.9.3 Support for PC-I 04/PCI, all CAN-IBXX0 and USB-to-CAN V2 interfaces
    • Added support for rx/tx thread priority, scheduling algorithm and cpu affinity for linux
    • Added support for rx/tx thread priority and cpu affinity for RTX
    • Added support for rx/tx thread priority for INtime
    • Added support for rx/tx thread priority and scheduling algorithm for QNX and VxWorks
    • Implemented filter support for all CAN-IBxx0, USB-to-CAN V2, USB-to-CAN compact USB-to-CAN II, XC-16/PCI and XC-16/PCIe interfaces for more information see ECIDRV_CtrlSetAccFilter(), ECIDRV_CtrlAddFilterIds() and ECIDRV_CtrlRemFilterIds()
    • Added RTX64 2013 and RTX64 2014 support
    • Added new USB-to-CAN V2 firmware to EciDeviceAdmin. Fixes problems with virtual machines.
    • Python examples implemented for Linux OS
    • BCI Single Transmit Try compatibility mode added for XC-16/PCI, XC-16/PCIe and CAN-IBxxx interfaces
  • Version: 1.8.2011.0, Date: 2014/03/27, OS: Linux i386 and Linux amd64
    • Added support of USB-to-CAN V2. Requires libusb-1.0
    • Support of linux kernels with versions greater than 3.10
  • Version: 1.7.1582.0, Date: 2013/04/16, OS: Linux i386 and Linux amd64
    • Fixed a problem in firmware download for iPC-I XC16/PCI(e) series interfaces while opening the ECI interface.
    • Updated firmware to 0.3.3.0 for iPC-I XC16/PCI(e) series interfaces to support CAN Single Shot Mode. For these interfaces single shot mode cannot be combined with CAN self-reception. Due to a hardware problem this feature is only available to iPC-I XC16/PCI(e) series interfaces with an Infineon XC16 micro controller with step "BA" or later.
  • Version: 1.7.1534.0, Date: 2013/02/21, OS: INtime 4.20.12200, INtime 5.12.12321 and RTX 2012
    • Added support for following interfaces: PC-I 04/PCI, CAN-IB1x0/PCIe series, iPC-I 320/PCI, iPC-I XC16/PCI, iPC-I XC16/PMC and CAN-IB2x0/PCIe series.
    • Added support of MSI (Message signaled interrupts) for all CAN-IBxxx/PCIe Interfaces when running under INtime or RTX.
    • Fixed a problem in ECI Device Administration Tool where a real-time operation system could stop the flash process due to exceeding thread spin control threshold.
  • Version: 1.7.1395.0, Date: 2012/12/12, OS: QNX x86
    • Added QNX x86 support for following interfaces: PC-I 04/PCI, CAN-IB1x0/PCIe series, iPC-I 320/PCI, iPC-I XC16/PCI, iPC-I XC16/PMC and CAN-IB2x0/PCIe series.
  • Version: 1.7.1340.0, Date: 2012/11/12, OS: Linux i386 and Linux amd64
    • Added support of iPC-I 320/ISA and iPC-I 320/104.
    • Added support of PC-I 04/104. This driver supports all memory mapped SJA1000 controller interfaces e.g. PC-I 03/ISA, but all interfaces will be named PC-I 04/104.
    • Added ECI Device Administration Tool to maintain the device code of the supported IXXAT interfaces. For more information see ECI Device Administrator.
  • Version: 1.6.1249.0, Date: 2012/09/07, OS: Linux i386 and Linux amd64
    • Added support of CAN-IB130/PCIe 104 and CAN-IB230/PCIe 104.
    • Added support of PC-I 04/PCI.
    • Added support of iPC-I XC16/PCIe.
    • Updated iPC-I XC16/PCI firmware to 0.3.2.4 to support LIN on PMC interfaces.
    • Improved CAN TX performance of CAN-IB2x0/PCIe (104) by reducing time needed to send a single CAN message.
    • Updated CAN-IB200/PCIe firmware to 0.3.7.0 which fixes some minor CAN error frame timestamp problems and improves RX performance by reducing time needed to put a single CAN message into the RX FIFO.
    • Added ECI-API function ECIDRV_CtrlGetStatus() to readout controller configuration, current controller status and current bus load (if supported by interface).
  • Version: 1.5.1112.0, Date: 2012/03/08, OS: Linux i386 and Linux amd64
    • Added support of CAN-IB100/PCIe and CAN-IB120PCIe Mini.
    • Updated CAN-IB200/PCIe firmware to 0.3.4.0 which fixes some minor LIN problems.
  • Version: 1.5.1021.0, Date: 2011/12/04, OS: Linux i386 and Linux amd64
    • Added support of CAN-IB200/PCIe.
    • Fixed a problem in LIN slave buffer update handling which may have an effect on already implemented ECI API programs. For further information please refer to ECI API LIN Support.
    • Bug in ECIDRV_CtrlOpen fixed in Linux version: Crash or deadlock may have occurred when driver open failed and the cleaning routines where called.
  • Version: 1.4.916.0, Date: 2011/04/18, OS: Linux i386 and Linux amd64
    • Added full support of LIN Controller available on USB-to-CAN II automotive and iPC-I XC16/PCI with LIN extension board.
    • Added openSUSE support.
    • Bug in Linux Kernel Module fixed: Controller could not be opened if multiple cards and multiple controllers where used.
    • Bug in OS_<func> API fixed. OS_<func> functions could not be used from C-programs because symbols where exported in C++ name mangling.
    • Changed ECI API Demo from C++ to C-code.
    • Some minor fixes in online help.
  • Version: 1.3.830.0, Date: 2010/11/19, OS: Linux i386 and Linux amd64
    • Added experimental support of LIN Controller available on USB-to-CAN II automotive and iPC-I XC16/PCI with LIN extension board.
    • Bug in Kernel Module fixed: Using another card together with an iPC-I320/PCI interface did not work because of an PLX9030 bug
    • Added CAN and LIN introduction to online help.
