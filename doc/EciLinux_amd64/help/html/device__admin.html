ECI Device Administrator

Device Code Maintenance

The ECI Device Administrator is a tool to maintain the device code of all supported IXXAT interfaces. This especially includes the CAN-IBxx0/PCIe series which are based on an FPGA design and thus can easily upgraded.

Device Information

The ECI Device Administrator can also be used to show information to all supported IXXAT interfaces. By setting --outputlevel=3 ECI Device Administrator shows very detailed information to the device and its controller including the count, type, bus coupling and features.

Command Line Arguments

Usage: 'EciDeviceAdmin [options]'

Option

Description

-h,
--help

Show this help and exit.

-v,
--version

Show version and copyright information and exit.

-o LEVEL,
--outputlevel=LEVEL

Set EciDeviceAdmin output level to [0..9] where 0 is no output and 9 is verbose output.
0=silent, 1=default, 2=verbose, 3=very verbose, 4=debug, 5=memory dump, 6=function call

-s,
--showonly

Show information only for all or specified interfaces without updating them. If this option is set, all flash update dependent options will be ignored.

-a,
--autoupdate

Enable automatic update for all or specified interfaces. This option may not be combined with '--showonly'.

-i TYPE [TYPE ...],
--interface=TYPE[,TYPE,...]

Limit EciDeviceAdmin to specified interface types. Several interfaces can be specified at once. The interface type is the hexadecimal number of the device driver. e.g.: 105 for iPC-I XC16/PCI

--simulatewrite

Enable write simulation mode. Flashing of the device is simulated. The flash memory remains untouched, only the verify step is really performed. This switch can be used to check the consistency of a flash firmware file.

--forceupdate

Enable forced update. Flashing of the device is performed, even if the version numbers match and no update is necessary. This option may not be combined with '--showonly'.

--forceverify Enable forced verify. Verifying the flash content is performed even if the version numbers match. In case verifying fails the device is updated. This option may not be combined with '--showonly'.
