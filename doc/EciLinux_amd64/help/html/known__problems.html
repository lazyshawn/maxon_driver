Known Problems

PC-I 04/PCI and iPCI-I 320/PCI

There is a known problem when having installed two or more PC-I 04/PCI and/or iPCI-I 320/PCI interfaces. If the PCI register memory area is mapped to a 0x80 boundary the driver will not be able to access it. On Linux the ECI kernel module then will issue following message 'ECI: PCI Region 3 not available, you need to update the EEPROM of your card.'
On other operation systems trying to open this interface will fail. This problem can be fixed by updating the EEPROM of the interface. Therefore the ECI Device Administrator can be used.

Update steps:

  • Shut down the PC and remove all PCI interfaces except one IXXAT PC-I 04/PCI or iPCI-I 320/PCI.
  • Restart the PC and change to the ECI install folder "/Eci<OS>/bin/release"
  • Start the EciDeviceAdmin without option. If an update of the EEPROM is required it will ask you 'Perform Update? : Flash EEPROM [y/n]'. Accept this question by typing 'y'.
  • Perform the last three steps for all PC-I 04/PCI and iPCI-I 320/PCI interfaces.
  • After a successful update of all interfaces, the ECI Linux kernel module should show following message for some interfaces: 'ECI: Using PCI Region 3' and all interfaces should work correctly. You can also check if all interfaces work correctly by starting EciDeviceAdmin. It should show information to all available IXXAT PC-I 04/PCI and iPCI-I 320/PCI interfaces.

CAN-IB2x0/PCIe (104) (Linux only)

There is a known problem with the CAN-IB2x0/PCIe series. When restarting the ECI Linux kernel module driver the CAN-IB2x0/PCIe interfaces cannot be accessed anymore. To fix this problem restart your PC.

ECI Device Administrator (RTX only)

RTX does not support console input. Thus the ECI Device Administrator asumes "NO" as an answer when it asks if an update should be performed. To update interfaces under RTX please use the command line switch "--autoupdate".

USB-to-CAN compact/USB-to-CAN II

It sometimes happens that USB-to-CAN compact/USB-to-CAN II is not working properly if it is connected to an usb3 port. To avoid this issue plug it to an usb2 port or disable usb3 functionality in your BIOS.

iPC-I XC16/PCIe (Linux only)

The interface card iPC-I XC16/PCIe is not recognized on kernel version 3.13.0-25. If you face this issue update your kernel to a newer version.

RTX64 2013

RTX64 2013 has problems loading the shared libraries. This results in a green screen. If such an issue occurs use the RTX64 2013 Update 2 Version of the RTX64 2013 OS.
